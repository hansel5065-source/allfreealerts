# AllFreeAlerts — Project Guide

## What This Project Is

A portfolio of 8 free-stuff aggregator websites, with **AllFreeAlerts.com** as the flagship. Each site has a scraper that pulls deals from dozens of sources, a static site that displays them, and automated daily workflows that keep everything fresh.

The goal: build passive income through ad revenue, affiliate links, and email subscribers by providing real value — aggregating free stuff people actually want.

## Daily Workflow — How It All Works

Every day at **11am ET**, GitHub Actions runs this pipeline:

1. **Scrape** → `scraper.js` pulls deals from 17 sources (sweepstakes, freebies, settlements)
2. **Dead link check** → `check_links.js` removes expired/broken deals
3. **QA new entries** → `qa_new_entries.js` validates links, removes middlemen, flags suspect items
4. **Email QA report** → sends removal/flag report to contact@allfreealerts.com via Brevo
5. **Generate RSS** → `generate_rss.js` creates the RSS feed
6. **Generate social images** → `ig_fb_autoposter.js --dry-run` creates deal-specific branded PNGs via Puppeteer
7. **Deploy** → pushes updated site to Cloudflare Pages (allfreealerts.com)
8. **Post to Instagram** → carousel post with 3 swipeable deal images (one per category)
9. **Post to Facebook** → 3 photo posts with deal images
10. **X/Twitter** → NOT automated (API costs $100/mo). Instead, generate 5 tweets + images daily for manual paste into Buffer

The workflow file: `.github/workflows/daily-update.yml`

## Critical Rules

### QA — NEVER Skip This
- **ALWAYS** use `--dry-run` first before any live post
- Verify: emojis render, images match captions, no undefined/null, image URLs accessible
- **NEVER** post test content to live accounts
- If user says "test it" → generate and preview locally, confirm with user, THEN post
- Past incident: broken emoji posts + generic images went to live @allfreealerts IG

### Middleman & Dead Link Rules
- **Middleman = link URL points to an aggregator**, NOT the original source
- We SCRAPE from aggregator sites (contestgirl, hip2save, etc.) to FIND deals — that's fine
- But the deal LINK must go to the original source (brand site, gleam.io, claim form, etc.)
- **Settlements are EXEMPT** — settlement aggregators (topclassactions, classaction.org) link to real claim forms and are legitimate
- `qa_new_entries.js` checks new entries daily; `check_links.js` checks older entries for dead links
- Shortlinks/trackers (freebi.es, trk.adbloom.co, etc.) are treated as middlemen
- Both scripts run automatically in the daily GitHub Actions workflow

### Image-Caption Sync Bug (Fixed)
- The script picks random deals each run. If dry-run and live-run pick different deals, captions and images mismatch
- Fix: `ig_fb_autoposter.js` now deploys images to Cloudflare BETWEEN generating and posting
- Always verify this flow works: generate → deploy → wait for CDN → post

### SEO Rules
- **Every HTML page MUST have:** `<meta name="robots" content="index, follow">` and `<link rel="canonical" href="https://allfreealerts.com/PAGE">`
- **Canonical URL is always `https://allfreealerts.com/`** (no www, HTTPS only)
- **www redirects to non-www** via `site/_redirects` file on Cloudflare Pages
- **Never add `noindex`** to any page unless explicitly asked
- **When creating new pages**, always include both the robots meta tag and canonical link in the `<head>`
- **Sitemap** (`site/sitemap.xml`) must be kept up to date when adding/removing pages
- Past incident (2026-03-27): Google detected `noindex` on homepage, likely from Cloudflare Pages preview deploy. Fixed by adding canonical tags + www redirect + requesting re-indexing

### Design Rules
- **Palette:** Teal (#0ABAB5) + Orange (#FF6F3C) + Gold (#F5A623)
- **Light theme only** — no dark mode, no purple
- **Never show source names** on any website
- **Never use the word "scan"** — use "find", "discover", "look for"
- **Tagline:** "Free Stuff, Found For You"
- **Copy style:** Casual and clear, not salesy
